# Dot Fossil Lab — 공룡 화석 부위 맞추기

닷패드(DotPad 320, 60×40) 촉각 학습 게임. 부위 맞추기 퀴즈에 더해 **공룡의 생애 · 화석이 만들어지는 과정 ·
실제 발굴 방법론**까지 폭넓게 흥미를 붙일 수 있도록 5개 모드로 구성했다.
React + Vite + TypeScript, 독립 폴더 빌드.

## 게임 구성

| 모드 | 내용 |
|---|---|
| 화석 탐색 학습 | 부위 5종(갈비뼈·발톱·척추뼈·다리와 발·부분 골격)을 닷패드로 출력, 촉각 힌트·공룡 지식 음성 안내 |
| **화석 신호 추적** (신규) | 《닷 포실 헌터》 MVP 루프 — 레이더 탐색 → 화석 발견 → 조사 → 후보 3장 비교 → 복원 → 도감 등록 (아래 상세) |
| 발굴 체험 | 탐침(25% 밀도) → 붓질(50%) → 정밀 추출(100%) 3단계로 실제 발굴 방법론을 체험 — 왜 서두르면 안 되는지, 왜 붓을 쓰는지까지 |
| 화석이 되기까지 | 매몰 → 퇴적 → 광물화 → 융기 → 침식 → 발견, 6단계 지층 단면 촉각 다이어그램으로 화석 형성 과정을 서사로 안내 |
| 공룡 이야기 | 5개 공룡의 시대·서식지·먹이·행동 지식 + 스쿨버스(12m)와 비교하는 크기 비교 촉각 막대 |
| 퀴즈 1 · 부위 맞추기 | 패턴을 만지고 4지선다로 부위 이름 맞추기 (형태 변별) |
| 퀴즈 2 · 주인 찾기 | 부위 → 주인 공룡 연결 (공룡 10종 보기 풀) |
| 화석 도감 | 학습/발굴/퀴즈/이야기/신호추적 진행 배지, 패턴 재출력, localStorage 저장 |

## 화석 신호 추적 — 《닷 포실 헌터》 MVP 루프

사용자가 공유한 설계 문서(《닷 포실 헌터: 잃어버린 공룡을 찾아라》)의 MVP 루프를 그대로 구현했다:
**레이더 탐색 → 화석 발견 → 조사 → 후보 카드 비교 → 복원 → 도감 등록**. 문서가 제안한 홈 3버튼 재편,
난이도 3단계, 조각 3종 수집 시스템은 범위가 커서 이번 라운드에는 넣지 않았다(아래 "이번에 넣지
않은 것" 참고).

1. **레이더 탐색** — 방향키로 가상 발굴지 이동. 근접도는 음높이(사운드)로, **방향은 닷패드 가장자리
   핀 클러스터(좌/우/상/하)로 전달**한다. 스테레오 패닝 대신 촉각 위치를 쓰는 이유는 여전히 헤드폰
   없는 교실 스피커 호환성 때문. 거리 카테고리가 바뀔 때만 "오른쪽으로 3칸 더 가 보세요" 같은 방향
   힌트를 음성으로 준다(매 이동마다 말하면 소음이 되므로).
2. **화석 발견** — 근처에서 엔터를 누르면 화석이 드러나고 손끝 조사로 넘어간다.
3. **조사** — 화석마다 실제 패턴을 분석해 도출한 참/거짓 과제 하나씩(아래 표). 오답이어도 끝나지
   않고 힌트를 주고 재도전시킨다(화면에도 표시).
4. **후보 카드 비교** — 실루엣 자산이 있는 5종(브라키오·벨로키랍토르·스피노·파라사우롤로푸스·
   안킬로) 중 정답 1 + 오답 2를 무작위 구성. 좌우로 카드를 넘기면 공룡별 시그니처 사운드 +
   전신 실루엣이 닷패드에 출력된다. 오답 선택도 실패로 끝내지 않고 특징 대조 힌트를 주고 재도전.
5. **복원** — 3틱에 걸쳐 에너지가 모이는 연출(사운드+레이더 반지름 축소) 후 엔터로 확정.
   **실패 페널티 없음** — 타이밍이 늦어도 항상 성공하고, 다만 늦게 눌러도 되니 편하게.
6. **도감 등록** — 복원한 공룡은 화석 도감에 "복원 완료" 배지로 남는다.

### 공룡 전신 실루엣 — 제작 기준

문서가 제시한 기준을 그대로 따랐다: 모두 옆모습, 원근 없음, 머리는 오른쪽으로 통일, 세부 묘사보다
큰 외곽선 우선, 종별 특징 과장, 최소 2핀 두께, 가장자리 2~3핀 여백. `src/dotpad/silhouette.ts`의
두꺼운 선(`stroke`)·꺾은선(`polyline`)·채운 원(`blob`) 세 프리미티브로 `src/data/dinoSilhouettes.ts`
에 5종을 코드로 작성했다(스캔한 DTMS 자산이 아니라 절차적 생성 — 특징: 브라키오=아주 긴 목,
벨로키랍토르=긴 꼬리+낫 발톱, 스피노=등 돛, 파라사우롤로푸스=휜 볏, 안킬로=낮은 몸+꼬리 곤봉).

#### 난이도 3단계

탐사 시작 시 항상 먼저 난이도를 고른다(라운드가 바뀌어도 유지, 도감/다른 화면 이동 후 재입장하면
다시 선택). 새 자산을 늘리지 않고 **정보 노출량과 시도 횟수**만으로 난이도를 구성했다 — 몸체가
비슷한 "혼동용 공룡"을 추가하는 대신, 있는 자산 안에서 힌트 텍스트 분량과 재도전 기회를
줄여 난이도를 만든 것.

| 난이도 | 카드 설명 | 조사 시도 횟수 | 콤보 |
|---|---|---|---|
| 탐험가 | 전체 특징 문장 | 무제한 | 없음 |
| 연구원 | 핵심 단서 한두 단어만 | 2회(소진 시 정답을 알려주고 다음 단계로, 페널티 없음) | 없음 |
| 고생물학자 | 연구원과 동일(단서 최소) | 2회 | 연속 정답 3·5·10회에 콤보 음성 안내 |

## 이번에 넣지 않은 것 (다음 단계 후보)

- 홈 화면 3버튼(탐험 시작 / 공룡 연구 도감 / 훈련 연구소) 재편 — 기존 학습·퀴즈 모드를
  하위 메뉴로 옮기는 IA 변경은 범위가 커서 보류, 지금은 기존 평면 메뉴 유지
- 공룡별 조각 3종(몸/특징/발자국) 완성 도감 카드, 완성 시 울음소리·희귀 카드 해금
- 카드 후보를 5종 이상(트리케라톱스 등 소유자 없는 종)으로 확장, 혹은 "비슷한 체형" 결정 후보 —
  지금은 실루엣 자산이 있는 5종 안에서만 후보를 뽑음(자산 예산 안에서 모든 카드가 실제 실루엣을
  갖도록 하는 선택)
- 손끝 타이밍 캐치/복원 미니게임의 엄격한 성공/실패 판정 — 블루투스 지연 우려로 항상 성공,
  타이밍은 "보너스 문구"로만 반영

## 촉각 설계 — 큰 단계로만 구별

발굴 체험의 탐침/붓질 단계는 처음엔 "실제 화석 윤곽만 남기기(모폴로지 아웃라인)"로 시도했으나,
DTMS 실측 패턴이 이미 얇은 선 형태라 **윤곽 ≈ 전체 패턴의 80~97%**가 되어 버려 손끝으로 단계 차이를
구별할 수 없었다. 대신 좌표 기반 결정적 서브샘플링(`quarterDensity`/`partialOf`)으로 바꿔
**탐침 25% → 붓질 50% → 추출 100%**의 확실한 밀도 차이를 보장했다(`src/dotpad/tactileGen.ts`).
화석이 되기까지의 6단계도 매몰량(coverage rows)과 화석 노출 밀도를 큰 폭(0→4→14→26→18→8→0行,
hidden→faint→partial→full)으로만 바꿔 동일 원칙을 지켰다.

## 임베드 규격 (Embed Spec) 준수

- **1. `?embed=1`** — 브랜딩 헤더 숨김, 게임 보드·핵심 컨트롤만. 홈 메뉴에 "게임 나가기" → `postMessage {type:'exit'}` (Back 대체)
- **2. `?preview=0`** — 핀 플레이트(화면 시뮬레이션) 미렌더. `embed=1`이면 기본 off, `?preview=1`로 강제 표시 가능. 실제 닷패드 출력은 항상 유지
- **3. 반응형** — 고정 픽셀 레이아웃 없음, 보드 중앙 배치, 좁은 창/낮은 창 대응
- **4. postMessage** — `{source:'dotarcade', type:'ready'|'resize'|'exit'}`. resize는 ResizeObserver로 높이 통지
- **5. 독립 폴더** — `vite.config`의 `base:'./'`, 모든 자산 상대경로. `games/<game>/index.html?embed=1&preview=0` 형태로 임베드 가능
- **6. 닷패드 키 표준** — F1/F2 행 이동 · Pan L/R 열 이동 · F3 선택 · F4 표시 · Pan All 음성 재생. 하드웨어 키를 DOM keydown으로 재발행해 키보드 핸들러와 단일화
- **7. 음성/접근성** — 공용 표준 `TW_TTS`(dot-games-host tts.js) 사용, speechSynthesis는 오프라인 개발 폴백만. 키보드는 방향키·Enter·F1~F4만으로 전 기능 조작. 모든 화면 순환 목록 끝에 "홈으로" 항목 → 닷패드 단독으로도 전 화면 이동 가능

## 개발·배포

```bash
npm install
npm run dev       # 로컬 개발 (Web Bluetooth는 localhost에서 동작)
npm run build     # dist/ 생성 — 이 폴더를 그대로 배포
```

- Web Bluetooth는 **HTTPS 필수** (GitHub Pages / Vercel OK). iframe 임베드 시 부모가 `allow="bluetooth"` 부여(TW가 처리).
- 점자 텍스트 라인: braillify.kr 점역 API(약자 포함) 우선, 실패 시 로컬 기본 테이블 폴백.
- 제출 전 확인: 실행 주소 끝에 `?embed=1` 붙여 음성·키보드 조작 점검.

## 알려진 버그 수정 이력

`useDotPad()`가 렌더마다 새 객체를 반환해서, `updateProgress`(localStorage 저장)로 상위 컴포넌트가
리렌더될 때마다 자식 화면의 `pad` prop 참조가 바뀌고, 그 참조에 의존한 마운트 이펙트가 재실행되어
**진행 상황을 기록하는 순간 화면이 처음 상태로 리셋**되는 문제가 있었다(학습 상세 듣기, 퀴즈 정답,
발굴 완료 등 모든 진행도 기록 지점에서 재현). `useDotPad`의 반환 객체를 `useMemo`로 안정화해 해결.

## 촉각 패턴 출처

`src/data/parts.ts`의 600-hex 패턴은 dot-fossil 저장소의 실측 `.dtms` 자산
(rib / claw / vertebra / legfoot / partialSkeleton). 별칭 패턴(shell=rib, tooth=vertebra,
fish=partialSkeleton)은 촉각 변별이 불가능하므로 문항에서 제외했다. 발굴 체험·화석 형성 과정의
패턴은 `src/dotpad/tactileGen.ts`의 결정적 함수로 생성한다(난수 없음, 항상 동일 결과).
